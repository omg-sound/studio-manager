# 프로젝트 목록 — 지메일식 한 줄 행 + 밀도 토글(좁게/넓게) 설계

> 2026-07-14 · 사용자 제안: "카드 높이를 한 줄로 해서 아티스트 이름만 보고 식별하기 좋게. 좌측=아티스트·제작사·프로젝트명, 우측=다음 세션·디데이."
> 후속 합의: 밀도 **토글**(좁게/넓게)을 둬서 훑기(좁게)와 판단(넓게)을 둘 다 지원한다.

## 1. 결정 사항(사용자 확정)

| 질문 | 결정 |
|---|---|
| 행 단위 | **행 = 프로젝트**(지금과 동일). 탭도 기존 3탭(진행 중 / 청구 필요 / 완료) 유지 |
| 펼침(세션·작업 + 완료 토글) | **유지** — 행 오른쪽 토글로 축소 |
| 우측 정보 | **탭마다 그 탭에 중요한 것만** |
| 넓게 모드 구성 | **지금 카드 내용 그대로, 단 카드 테두리 없이 구분선 목록으로** |
| 기본 밀도 | **좁게(한 줄)** |

**보류(별도 과제)**: "프로젝트가 세션과 작업을 둘 다 담아 번거롭다" — 탭을 '진행중인 세션/진행중인 작업'으로 쪼개는 안은 이번 스코프에서 제외(행 단위가 프로젝트로 유지되므로). 필요해지면 별도 설계.

## 2. 레이아웃

### 좁게(기본)
```
아이유          (주)이담 · 루나 1집 타이틀곡 '월광'          7월 20일 D-6   ⌄
태연            SM · 믹싱 의뢰                              7월 22일 D-8   ⌄
윤종신          미스틱 · 봄여름가을겨울                      8월 2일 D-19   ⌄
```
- **좌측 고정 열**: 아티스트(굵게·`truncate`, `sm:w-40`). 여러 명이면 `아이유 외 1`. 아티스트가 없으면 회사명, 둘 다 없으면 프로젝트명 승격(기존 `projectIdentity` 규칙 재사용하되 아티스트/회사를 분리 표기).
- **가운데**: `제작사 · 프로젝트명`(muted·`truncate`). 회사가 아티스트 본인 파생이면 회사 부분 생략.
- **우측(탭별)**: 진행 중=`다음 세션 날짜 + 디데이 pill`(현행 3단계 색 유지) / 청구 필요=`금액 + 청구 필요 N` / 완료=`작성일`.
- **행 끝 ⌄**: 펼침 토글. 닫힌 상태에는 카운트 요약이 없다(완전한 한 줄).
- **PM은 좁게에서 표시하지 않는다**(내 것만 보려면 '내 프로젝트만' 토글).

### 넓게
현재 카드의 정보 구성 그대로. 단 **카드 테두리·라운드 제거 → 구분선(`border-b`) 목록**:
- 좌측: 정체성(`아티스트 · 회사`, 굵게) / 프로젝트명 부제(다음 줄)
- 우측: 작성일 → PM → 다음 세션(또는 청구 필요 탭의 금액·배지)
- 행 하단: 카운트 요약(`예정 세션 N · 완료 세션 N · 곡·콘텐츠 N …`)
- 펼침 내용은 두 밀도 공통(세션·작업 목록 + 완료 토글).

### 모바일(<640px)
좁게도 2줄로 자연 래핑: 1줄=아티스트 + 우측값, 2줄=제작사·프로젝트명. 터치 타깃 44px 규칙(src.css 언레이어드 미디어쿼리) 유지.

## 3. 구현

### 3.1 마크업 — `<details>` 하나, DOM 한 벌
```html
<details class="proj-row" id="proj-7" [open]>
  <summary class="proj-summary">              <!-- 행 전체(클릭 여백·⌄ = 펼침) -->
    <a href="/projects/7?tab=sessions" class="proj-main">   <!-- 이름·제목 클릭 = 열기 -->
      <span class="proj-artist">아이유</span>
      <span class="proj-sub">(주)이담 · 루나 1집…</span>
    </a>
    <div class="proj-meta">
      <span class="proj-created">2026-07-01</span>   <!-- 넓게 전용 -->
      <span class="proj-pm">PM 박수한</span>          <!-- 넓게 전용 -->
      <span class="proj-next">7월 20일 <b>D-6</b></span>   <!-- 진행 중 -->
      <span class="proj-amount">₩1,200,000</span>          <!-- 청구 필요 -->
    </div>
    <svg class="proj-chevron" />
    <span class="proj-counts">예정 세션 2 · 곡·콘텐츠 3</span>  <!-- 넓게 전용(접힌 상태에도 보임) -->
  </summary>
  <div class="proj-expand"> … 세션·작업 목록 + 완료 토글(현행 projectSummaryHtml) … </div>
</details>
```
- **DOM은 한 벌**이고 밀도는 CSS로만 전환한다(두 벌 렌더 금지 — HTML 두 배).
- **클릭 규약(사용자 확정)**: **행 어디를 눌러도 펼침**(⌄ 버튼과 동일) / **제목(아티스트·제작사·프로젝트명 텍스트)만 상세로 이동**(`projectRowHref` — 청구 필요=청구 탭, 진행 중=세션/곡). `<summary>` 안의 `<a>`가 제목 영역이고 나머지 여백·우측값·⌄는 토글. JS 0·CSP 안전.
- `?open=<id>`로 돌아왔을 때 그 행을 펼치는 현행 동작(`open` 속성)·완료 토글 폼·app.js 스크롤 보존은 **그대로**.

### 3.2 밀도 토글 — 테마 토글과 동일 패턴
- **CSS 기본 = 좁게**. 넓게는 `:root[data-density="comfy"]` 오버라이드(`src.css`). → 기본값엔 속성이 없어 **깜빡임(FOUC) 없음**.
- app.js: `[data-density-toggle]` 클릭 → `localStorage["density"]` 저장 + `<html data-density>` 갱신, 로드 시 저장값 복원(테마 코드와 같은 구조·같은 위치).
- 토글 버튼 위치: 프로젝트 목록 **탭바 오른쪽**(검색·'내 프로젝트만' pill과 같은 줄), 라벨은 현재 상태 반대('넓게'/'좁게').
- 프로젝트 목록에만 적용(다른 목록엔 영향 없음 — 스코프 클래스 `.proj-row` 하위에서만 반응).

### 3.3 서버·데이터
`listProjects`·`splitProjectTabs`·정렬·`listProjectSummaries`·`projectRowHref` **무변경**. 바뀌는 건 `projectListRow`(views.projects.js) 마크업과 `src.css` 밀도 레인, app.js 토글 IIFE, 목록 라우트의 토글 버튼 렌더뿐이다.

## 5. 리스크 · 구현 중 발견

- **`<summary>` 안의 `<a>`** — 실브라우저 검증 결과 정상(링크 클릭=이동, 나머지=토글).
- ⚠️ **함정(실제로 밟음)**: `<a class="proj-main">`을 `flex: 1 1 auto`로 두면 **링크가 행의 남는 여백까지 차지**해 여백 클릭이 상세로 이동한다(= '행 클릭=펼침' 규약 파괴). 링크는 `flex: 0 1 auto`(내용 폭)로 두고 우측값(`.proj-meta`)을 `margin-left:auto`로 밀어야 한다. 넓게 모드도 `display:block`이면 같은 문제 → `inline-block`.
