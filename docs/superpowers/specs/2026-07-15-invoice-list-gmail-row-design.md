# 청구 목록 지메일식 한 줄 행 — 설계 (2026-07-15)

## 배경 / 문제

청구 목록(`/invoices`)은 청구서 하나가 **카드 한 장**(제목 + 배지 + 부제 + 금액 + 상태 버튼 2개, ~110px)이라
한 화면에 6~8건밖에 안 들어온다. 그런데 목록에서 실제로 하는 일은 두 가지뿐이다.

1. **식별** — "이거 누가 내는 청구지?" → 지금 카드의 제목("예린 청구")은 **프로젝트명 + '청구'** 라 정작
   결제 주체(청구처)가 부제 뒤쪽에 묻혀 있다.
2. **처리** — 계산서 발행 완료 / 입금완료 토글.

프로젝트 목록은 2026-07-14에 이미 지메일식 한 줄 행 + 밀도 토글로 재설계했다(같은 문제·같은 해법).
청구 목록도 같은 골격으로 맞춘다.

## 결정 (사용자 확정)

- **좌측 = 청구처 · (제작사 · 아티스트)** — 결제 주체를 첫 열로 승격.
- **우측 = 금액 + 상태 토글 2개**(짧은 pill: `계산서`/`입금`. 켜짐=색, 꺼짐=흐림. 둘 다 클릭 토글).
- **행 클릭 = 펼침**(청구 항목·소계·VAT·PDF), **제목 클릭 = 청구 상세**. 프로젝트 목록과 동일 규약.
- 밀도(좁게/넓게)는 **프로젝트 목록과 같은 설정**(`localStorage["density"]`)을 공유한다.

## 행 구성

### 좁게(기본, 42px 한 줄)

```
[청구처(고정 폭)] [제작사 · 아티스트] ······ [금액] [✓계산서][−입금] [⌄]
 (주)도너츠컬처    도너츠컬처 · 진혁          ₩220,000
```

- **청구처(`payerName`)** = 굵게, 고정 폭(`w-40`, 프로젝트 목록의 아티스트 열과 동일 톤).
  개인 청구처는 `본명 (활동명)` 병기(기존 `payerName` 그대로).
- **부제** = `제작사 · 아티스트`. 중복 조각은 생략한다:
  - 제작사 == 청구처면 제작사 조각 생략(같은 이름이 두 번 나오지 않게)
  - 아티스트가 여러 명이면 `첫 아티스트 외 N`
  - 둘 다 없으면 프로젝트명, 프로젝트도 없으면(수동 발행 잔존분) 청구 제목
- **금액** = `formatKRW(inv.amount)` (VAT 포함 총액, 현행과 동일)
- **상태 pill 2개** = `taxToggleButtons`의 **compact 변형**(라벨 `계산서`/`입금`, `btn-xs`).
  권한(`isInvoicer` = 치프·대표)이 없으면 **읽기 배지**(`invoiceBadge`)로 대체.
- **⌄** = 펼침 셰브런.

### 넓게 (`:root[data-density="comfy"]`, 같은 DOM에 CSS만 전환)

```
(주)도너츠컬처                                   ₩220,000  [✓계산서 발행 완료][−입금완료]
도너츠컬처 · 진혁 · 도너츠컬처 진혁 청구          OMG-202607-014 · 2026-07-14
```

- 청구처/부제 2줄, 부제에 **청구 제목**까지 붙는다.
- 우측 아래 줄에 **청구번호 · 발행일**(넓게 전용 — `.inv-comfy-only`).
- 상태 pill의 라벨이 **긴 형태**(`계산서 발행 완료`/`입금완료`)로 보인다(CSS로 짧은/긴 라벨 span 교체).

## 데이터

`listInvoices`(src/data/invoices.js)의 SELECT에 프로젝트 조인은 이미 있다(`LEFT JOIN projects p`).
**두 컬럼만 추가**한다.

```sql
p.production_company AS project_production,
p.artist            AS project_artist
```

새 쿼리·N+1 없음. 나머지(`client_name`=payer 표시, `payer_kind`, `tax_status`, `amount`)는 현행 그대로.

## 펼침(행 클릭) 본문

**`invoiceExpandBody`를 그대로 재사용**한다(프로젝트 청구 탭에서 쓰는 그 함수 — 청구처 정보 카드,
발행일, 청구 항목, 소계·할인·VAT·총액, PDF 3종, 삭제). 목록에서는:

- `returnTo` = `/invoices?tab=…&open=<id>`(현재 탭·검색 유지)
- 상태 처리(계산서·입금)는 **행 우측 pill**이 담당하므로 펼침 본문엔 넣지 않는다(현행 그대로 — 프로젝트
  청구 탭도 처리 컨트롤 없음).

## 클릭 규약 (프로젝트 목록의 함정 그대로 적용)

- 행 = `<details>` + `<summary class="row-link">` → **행 어디를 눌러도 펼침**.
- 제목(`<a class="inv-payer">`)만 상세로. ⚠️ 제목 링크를 `flex:1`로 두면 **행 여백까지 링크가 먹어**
  '행=펼침' 규약이 깨진다(프로젝트 목록에서 실브라우저로 검출한 함정) → 링크는 내용 폭(`flex:0 1 auto`),
  우측 블록은 `margin-left:auto`.
- 상태 pill은 `<form>` 안의 버튼이라 **`<summary>` 안에서 클릭해도 펼침이 같이 일어나면 안 된다** →
  pill 컨테이너에서 `click` 시 `e.stopPropagation()` + `e.preventDefault()`는 쓰지 않고,
  **`<summary>`의 기본 토글을 막는 대신** pill을 `<summary>` 바깥(행의 형제)으로 두지 않는다.
  → 실제 처리: pill 폼에 `data-no-toggle`를 달고 app.js가 그 안의 클릭에서 `e.preventDefault()`를 걸어
  `<summary>` 토글만 막는다(폼 제출은 정상 진행). **실브라우저 검증 필수 항목.**

## 탭·검색·미수 합계

무변경. 3탭(발행 필요/발행 완료/입금완료)·검색·미수금 합계 카드·`capList` 더 보기 그대로.
`?open=<id>` 복귀(app.js `#inv-<id>` 스크롤)도 그대로 쓰되, 이제 그 행이 **펼쳐진 채** 복귀한다
(프로젝트 목록과 동일 — `openId`면 `<details open>`).

## 밀도 토글

프로젝트 목록의 `[data-density-toggle]`·`localStorage["density"]`를 **그대로 공유**한다(별도 키 없음).
청구 목록 헤더에도 같은 토글 pill을 노출 → 한쪽에서 바꾸면 두 목록 모두 반영.

## 모바일(<640px)

좁게에서도 2줄(청구처+금액 / 부제+상태 pill). 프로젝트 목록의 `@media` 규칙과 같은 방식.
390px 가로 오버플로우 0 실측.
