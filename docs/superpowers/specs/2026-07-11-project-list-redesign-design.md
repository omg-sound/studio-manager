# 프로젝트 목록 재설계 — 설계 문서

> 작성 2026-07-11. 목적: 프로젝트 목록을 "누가·무엇이 진행 중인가"에 시선이 모이도록 재편.
> 정체성 = **회사·아티스트**, 액션 = **다가오는 세션 / 청구 필요**. 프로젝트명·작성일·금액은 시선에서 내림.

## 배경 / 문제

현재 목록 카드는 위에서부터 `작성일(치프는 편집 가능한 date 입력) → 프로젝트명(가장 굵음) → 아티스트·회사·담당자(작게) → 다음 세션(작게) → PM·금액` 순이다.

- **작성일**이 최상단 + 편집 가능한 입력이라 "이 날짜에 뭔가 해야 할 것 같은" 잘못된 액션 신호를 준다.
- 정작 정체성인 **회사·아티스트**가 시각적 하위에 깔린다.
- **프로젝트명**이 최상위(가장 굵음)지만, 프로젝트명은 구분용 라벨일 뿐이다.
- 정렬이 작성일 최신순이라 "지금 진행 중·임박한 것"이 위로 오지 않는다.

## 결정 사항 (사용자 승인 완료)

1. **식별**: 회사·아티스트를 주(主), 프로젝트명은 부제.
2. **정렬(진행 중 탭)**: 다가오는 세션 임박순.
3. **작성일**: 카드에서 제거 → 프로젝트 상세에서만(치프 편집도 상세로 이동).
4. **PM**: 유지.
5. **완료 탭**: 같은 카드 스타일로 통일.
6. **금액**: 진행 중·완료 카드에서 제거(청구 필요 탭에서만 표시).
7. **탭**: 3탭 — 진행 중 / 청구 필요 / 완료.

## 탭 구성 (상호 배타 3그룹)

기존 `is_completed` 파생을 재사용한다(`content_cnt>0 && upcoming_cnt===0 && open_tasks===0`).

| 탭 | 조건 | 정렬 | 의미 |
|---|---|---|---|
| **진행 중** (`active`, 기본) | `!is_completed` | 다가오는 세션 임박순(`next_session_date` ASC, 없으면 뒤로) → 그 안 `created_at` DESC | 아직 세션·작업이 남음 |
| **청구 필요** (`billing`) | `is_completed && unbilled_cnt>0` | `created_at` DESC | 활동 끝, 청구 안 함 = 지금 처리할 액션 |
| **완료** (`done`) | `is_completed && unbilled_cnt===0` | `created_at` DESC | 청구까지 끝난 순수 아카이브 |

- 탭 라벨에 개수 병기: `진행 중 N` / `청구 필요 N` / `완료 N`. `청구 필요 N`은 N>0일 때 warning 톤 강조(선택).
- 각 탭이 비면 안내: "진행 중인 / 청구가 필요한 / 완료된 프로젝트가 없습니다."
- 정렬·분류는 모두 라우트(JS)에서 수행 — `listProjects`는 이미 `next_session_date`·`unbilled_cnt`·`is_completed`를 제공하므로 **새 SQL 쿼리 없음**.

## 카드 렌더링 (`projectListRow`, 탭별 변형)

공통 뼈대(위 → 아래):

```
정체성 줄        아티스트 · 회사                 ← 굵게(기본 크기). 시선 최상위
부제             프로젝트명                      ← 작게·회색. 없으면 줄 생략
(진행 중만)      ● 다음 세션 M/D · D-n           ← 다가오는 세션 없으면 줄 생략
PM               PM {담당 엔지니어}               ← 우측, 유지
──────────────────────────────────────
예정 N · 완료 N · 곡 N (작업 대기 N)          ⌄  ← 접힘 토글 바(카운트) / 펼치면 일정·곡 상세
```

**정체성 줄 규칙**
- 형식: `아티스트 · 회사`(회사 = `client_name`, 제작›소속›아티스트 파생).
- **중복 방지**: 회사가 아티스트 본인으로 파생된 경우(개인 아티스트, 회사 없음) → 아티스트만 표기.
- **여러 아티스트**: `아이유 외 1`로 축약(`p.artist` 콤마 목록 기준).
- **담당자(연락처)는 앞면에서 제외** → 펼침 상세/프로젝트 상세로. 회사·아티스트에 시선 집중.
- **폴백**: 회사·아티스트 둘 다 미정이고 프로젝트명만 있으면 → 프로젝트명을 정체성(주) 줄로 승격, 부제 생략.

**부제(프로젝트명)**: `text-sm text-muted`. 값 없으면 줄 자체 생략.

**앞면은 최소로 유지 — 카운트는 앞면이 아니라 접힘 토글 바에 둔다**(사용자 결정: 앞면 최소 + 펼침에 전부). 앞면은 정체성·부제·다음 세션·PM만.

**탭별 중앙 영역**
- **진행 중**:
  - 다음 세션 강조 줄: 색 단계 —
    - 오늘 = `● 오늘` 주황 강조
    - D-1~3 = `● 다음 세션 M/D · D-n` 파랑(primary) 강조
    - D-4~7 = 보통 색
    - **없음 = 줄 자체 생략**(사용자 결정 — "예정 세션 없음" 같은 문구도 표시하지 않음)
  - **금액 없음.** (진행 상태 카운트는 아래 접힘 토글 바로.)
- **청구 필요**:
  - `청구 필요 N` 배지(warning).
  - 금액(청구 규모 참고): `projectAmount(p)` 우측 표시(여기서만 금액 노출).
  - 다음 세션 줄 없음(완료라 없음).
- **완료**:
  - 정체성 + 부제 + PM만. 금액·다음 세션·배지 없음(순수 아카이브).

**접힘 상세(`<details>`) — 앞면 최소의 짝**:
- **토글 바(접힌 상태)**: 카운트 요약 `예정 N · 완료 N · 곡 N (작업 대기 N)` + chevron. **0인 항목은 생략**하고, **곡·콘텐츠가 없으면 "곡·콘텐츠 미정" 같은 표기 없이 그 부분을 생략**(사용자 결정). 셋 다 0이면 "세션·곡 상세" 같은 최소 라벨.
- **펼친 내용**:
  - **세션**: **다가오는 세션(오늘 이후)을 먼저** 날짜·시간·종류로 보여주고, 지난 세션은 그 아래 회색으로. 현재 `projectSummaryHtml`은 `session_date ASC` + 앞 8건만 잘라 **지난 세션이 앞을 먹어 다가오는 세션이 잘리는 약점**이 있으므로, 렌더 시 upcoming-우선으로 재정렬한 뒤 자른다(쿼리 변경 없이 JS 재정렬).
  - **곡·콘텐츠**: 현행 유지(곡별 아티스트·제목·작업자 + 작업 종류별 내역 `튠 1 · 믹싱 1 · 마스터링 1`).
  - 곡·세션 둘 다 없으면 "등록된 세션·곡·콘텐츠가 없습니다"(현행 유지 — 펼친 안쪽 안내는 유지, 접힌 토글 바 카운트만 생략).
- 탭 무관 동일 펼침(완료 탭도 같음, 기본 접힘). 네이티브 `<details>` 유지(무JS·CSP 안전).

**작성일 관련 제거**: 카드 상단 `dateRow`(치프 date 입력 / 일반 표시) 완전 제거. `projectListRow`의 `isChief`/`q`/작성일 인자 정리.

## 프로젝트 상세 — 작성일 편집 이동

- 치프의 작성일 인라인 수정을 **프로젝트 상세 메타**(`projectMetaCard`/`projectEditForm` 영역, 치프만)로 이동.
- 기존 라우트 `POST /:id/created-at`(`requireChief`) 재사용. 복귀 경로에 상세(`/projects/:id`)도 지원하도록 `return`/`from` 처리 추가(목록 복귀와 양립).
- 정렬이 임박순·작성일순이라 작성일 편집은 완료/청구 필요 탭 순서에만 영향(진행 중은 세션 임박순 우선).

## 영향 파일

- `src/routes/projects.routes.js` — GET `/` 3탭 분리·탭별 정렬·라벨·빈 상태·`tab` 파라미터(`active`/`billing`/`done`)·`searchBox` hidden tab·`capList` URL·`projectListRow` 호출 인자. `POST /:id/created-at` 복귀 경로.
- `src/views.projects.js` — `projectListRow` 탭별 카드 재구성·`dateRow` 제거·정체성 줄 헬퍼(중복 방지·외 N)·다음 세션 색 단계(없으면 생략)·접힘 토글 바 카운트(0·곡 없음 생략). `projectSummaryHtml` 세션 upcoming-우선 재정렬. 상세 메타에 치프 작성일 편집 필드.
- 데이터 레이어(`src/data/projects.js`) — **변경 없음**(기존 파생 필드 재사용).

## 예외 / 엣지 케이스

- 회사=아티스트 파생 중복 → 아티스트만.
- 아티스트 여러 명 → `외 N` 축약.
- 회사·아티스트 미정 → 프로젝트명 주 줄 승격.
- 프로젝트명 없음 → 부제 줄 생략.
- 진행 중인데 다가오는 세션 없음(예: 믹싱 대기) → 정렬상 진행 중 탭 하위, **다음 세션 줄 생략**(문구 없음).
- 곡·콘텐츠 없음 → 접힘 토글 바에서 곡 부분 생략(문구 없음).
- 검색(`q`) 시 3탭 개수·활성 탭 필터 유지(hidden `tab`).

## 검증 계획

- 단위/가드: 탭 분류 3그룹 상호 배타(진행 중/청구 필요/완료 합 = 전체) — 순수 함수로 분리해 테스트.
- 렌더: DEV_LOGIN 서버로 3탭 각각 200·카드에 회사·아티스트 굵게·금액 미표시(진행 중/완료)·청구 필요 탭 금액+배지 노출·작성일 미표시 확인.
- 정렬: 임의 세션 날짜로 진행 중 탭 임박순 정렬 확인, 세션 없는 프로젝트가 뒤로.
- 회귀: 기존 테스트 중 프로젝트 목록/탭 키(`active`/`done`)·`projectListRow` 참조가 있으면 `billing` 추가·금액 제거에 맞춰 갱신.
- 반응형: 320/360/390px 가로 오버플로우 0(정체성 줄 truncate).

## 범위 밖 (하지 않음)

- 곡·콘텐츠 작업에 실제 **마감일** 도입(현재 상태만 존재 — 별도 논의 필요).
- 대시보드 '청구 필요' 카드 구조 변경(현행 유지, 새 탭과 상호 보완).
- 완료/청구 필요 탭의 임박순 정렬(작성일순 유지).
